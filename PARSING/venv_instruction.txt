Виртуальное окружение (venv) - это инструмент, который помогает изолировать зависимости разных проектов. Оно полезно, когда работаешь с проектами в которых требуется использовать различные версий библиотек

К примеру:
Вы работаете над двумя проектами
    1. проект требует чтобы версия Django была 4.1
    2. проект требует чтобы верстя Django была 3.2
    
Если вы установите обе версий Django на компьютер, то они будут конфликтовать, так как какая то из них будет последней, и обе версии работать не будут


1)Команда для создания виртуального окружения:
    python3 -m venv my_project_env
    my_project_env - название папки, в которой будет создано виртуальное окружение
    (зачастую название указывают venv)

2)Команда для активации виртуального окружения:
    Для Windows:
    . my_project_env/Scripts/activate

    Для MacOs/Linux:
    1)source venv/bin/activate
    2). venv/bin/activate

3) Команда для деактивации виртуального окружения
    deactivate

4) Установка зависимостей:
    когда вы создали и активировали виртуальное окружение, вы можете скачать нужные вам библиотеки с нужными версиями при помощи команды:

    1) pip (pip3) install Django==3.2
    2) pip (pip3) install -r requirements.txt
        --- файл requirements.txt полезен когда у вас много зависимостей, и вам нужно хранить их в каком то файле

5) Команда для просмотра всех установленных зависимостей в виртуальном окружении
    pip freeze
